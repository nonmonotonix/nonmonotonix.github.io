Getting started with pages.github.com

2013-08-18

Last week's been a flashback to 1996 – people asking me how I'd publish a personal webpage. This post is for their convenience.
As a tight-fisted startup survivor these days I’m defaulting to pages.github.com. It’s free, fast, has no ads, you get to use it with your own domain and it forces you into the good habit of putting your site under source control. The latter is a system that keeps every version of your work and lets you jump back if desired. The tool used for this is called 'git', it's aimed at programmers, but thanks to Syntevo's brilliant Smartgit UI it very easy to get started. Free for non-commercial use, cheap otherwise.

The main drawback is that git not great at handling large or many changing binaries (non-text files) the performance of working with git will deteriorate, so if you plan on having a lot of images, host your images elsewhere and link to them from your HTML.

Steps

1. Create an account at Github

Privacy note: git / github are public repositories and every time you upload something, your email address is attached to it. I recommend you create a disposable email account (or use the email forwarding feature from your domain provider) for privacy and spam reasons.

2. Install Smartgit or the command-line Git. Both work across Windows, Linux and OS X.

3. Create a repository

Read this first, and decide if you want a project or a user/organization page. For the rest of this post, I'll assume you want a personal page. At Github, create a new repository (big green button) and name it username.github.io where username is your github username. In my case the repository is called 'nonmonotonix.github.io'. By default it lives at http://nonmonotonix.github.io on the web.

4. Create your content

Two options, either use the built in editor (A bit meh. Access via Settings - Automatic pages in your repository) or roll the HTML yourself. I'm assuming the latter.

  • Start by intializing your repository on your computer. In Smart git: Project -> Clone -> https://github.com/nonmonotonix/nonmonotonix.github.io.git (look for 'HTTPS clone URL' on your repo home page if you can't work out your setting) Remember the destination folder, that'll be the root directory of any further work.
  • In the root, add index.html. Additional content lives in subfolders or as a sibling.
  • In the root, add a text file called CNAME it holds the URL you want your site to appear under, "blog.nonmonotonic.org" in my case.
  • "README.md" and "LICENSE" should already exist - put any license/copyright notices in them. Github is the home of many open-source projects so make it clear what can and can't be reused by others.

    • Once that's done, select all your files in Smartgit. From the Toolbar, Commit (takes a snapshot of the selected files), then Push (publishes to your repo). It can take up to 10 minutes for github to make your upload available.
